=Features=

  • Export ABAP components from SAP systems to files
  • Import ABAP components from files to SAP systems
  • Uninstall ABAP components from SAP Systems
  • Activate components after import
  • Handle dependencies between components
  • Provide checksums for components
  • ... [Features Do you want to learn more ?]

=Framework= ZAPLink do not only provide features, but also a software architecture that allow (or try to) new features or enhancment to be made without having to change the framework. This will allow developer to have their own tools using ZAPLink features and being able to upgrade framework to the next version.

=Important notice= Because ZAPLink is a framework and not a test program/tools it use SAP devloppements concepts like :

  • Package
  • Application logs
  • Checkpoints Installing or using this framework will : # Create packages "ZAPLINK*" at the installation (you could choose $TMP) # Create application log object (ZAPLINK) and subobject which is part of SAP customizing task done with transaction SLG0. You may check that you have access to this transaction in order to be able to remove all ZAPLINK traces if required # Create checkpoint : ZAPLINK
